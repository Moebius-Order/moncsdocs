## üìö The Four Pillars

### 1. Foundational Theory

<div align="left">

**[üìñ Explore Foundational Theory ‚Üí](./foundations/README.md)**

</div>

The mathematical and theoretical foundations that underpin all of Computer Science.

**Core Topics**:
- **Mathematics for CS**: Discrete math, graph theory, combinatorics, probability
- **Algorithms**: Analysis, sorting, searching, divide-and-conquer, dynamic programming
- **Data Structures**: Arrays, trees, graphs, heaps, hash tables
- **Computational Theory**: Automata, formal languages, computability, complexity
- **Information Theory**: Entropy, coding, compression, cryptography basics

**Who Should Start Here**:
- Students beginning their CS journey
- Anyone preparing for technical interviews
- Developers wanting to strengthen algorithmic thinking
- Researchers exploring theoretical foundations

**Prerequisites**: High school mathematics, basic programming

---

### 2. Hardware & Architecture

<div align="left">

**[üîß Explore Hardware & Architecture ‚Üí](./hardware/README.md)**

</div>

The physical and logical structures of computer systems, from transistors to processors.

**Core Topics**:
- **Digital Logic**: Boolean algebra, logic gates, combinational and sequential circuits
- **Computer Organization**: CPU architecture, instruction sets, memory hierarchy
- **Memory Systems**: Cache, virtual memory, memory management
- **Advanced Architecture**: Pipelining, superscalar execution, multi-core processors
- **Embedded Systems**: Microcontrollers, real-time systems, IoT
- **Performance Engineering**: Optimization, profiling, benchmarking

**Who Should Study This**:
- System programmers
- Embedded systems developers
- Performance engineers
- Computer architects
- Anyone curious about how computers actually work

**Prerequisites**: Basic programming, foundational algorithms

<div align="left">

**[üíª Explore Software Paradigms ‚Üí](./software/README.md)**

Programming principles, software engineering practices, and system software development.

**Core Topics**:
- **Programming Paradigms**: Procedural, OOP, functional, declarative
- **Software Engineering**: Design patterns, SOLID principles, testing, agile
- **Compilers & Languages**: Lexical analysis, parsing, code generation, type systems
- **Operating Systems**: Process management, memory management, file systems, concurrency

**Who Should Study This**:
- Software developers at all levels
- Software architects
- System programmers
- Compiler engineers
- Anyone building software systems

**Prerequisites**: Programming experience, basic data structures

### 4. Systems & Networking

<div align="left">

**[üåê Explore Systems & Networking ‚Üí](./systems/README.md)**

Network protocols, distributed systems, databases, and security fundamentals.

**Core Topics**:

- **Distributed Systems**: CAP theorem, consensus, replication, consistency models
- **Database Systems**: Relational databases, SQL, NoSQL, transactions, indexing
- **Security Fundamentals**: Cryptography, authentication, authorization, web security

**Who Should Study This**:
- Backend developers
- DevOps engineers
- Database administrators
- Network engineers
- Security engineers
- Distributed systems architects

**Prerequisites**: Programming, basic OS concepts, algorithms

## üó∫Ô∏è Learning Paths

### For Beginners (Starting CS Journey)

**Recommended Order**:
1. Start with **Foundational Theory** basics
   - Discrete mathematics
   - Basic data structures (arrays, linked lists, stacks, queues)
   - Simple algorithms (sorting, searching)
2. Move to **Software Paradigms** fundamentals
   - Programming paradigms (procedural, OOP)
   - Basic software engineering
3. Explore **Hardware & Architecture** basics
   - Digital logic and binary systems
   - Basic computer organization
4. Progress to **Systems & Networking** fundamentals
   - Network basics and HTTP
   - Basic database concepts

### For University Students

**By Semester/Year**:
- **Year 1**: Foundational Theory (math, basic algorithms, data structures)
- **Year 2**: Software Paradigms (programming paradigms, OOP, basic OS concepts)
- **Year 3**: Hardware & Architecture + Advanced Algorithms
- **Year 4**: Systems & Networking (distributed systems, databases, security)

### For Self-Learners

**Project-Based Path**:
1. **Foundational Theory**: Learn enough algorithms and data structures for projects
2. **Software Paradigms**: Deep dive into programming and software engineering
3. **Systems & Networking**: Build web applications, APIs, and distributed systems
4. **Hardware & Architecture**: Explore as needed for optimization and embedded projects

### For Interview Preparation

**Focus Areas** (in order of importance):
1. **Foundational Theory** (70% of technical interviews)
   - Algorithms: Sorting, searching, dynamic programming, graphs
   - Data structures: Trees, heaps, hash tables
   - Complexity analysis
2. **Systems & Networking** (20%)
   - Database design
   - System design basics
   - Basic networking
3. **Software Paradigms** (10%)
   - Design patterns
   - OOP principles
   - Concurrency basics

### For Researchers

**Theoretical Depth**:
1. **Foundational Theory**: Deep dive into computational theory, complexity, information theory
2. **Hardware & Architecture**: Advanced architecture research topics
3. **Systems & Networking**: Distributed systems research, consensus algorithms
4. **Software Paradigms**: Programming language theory, type systems
